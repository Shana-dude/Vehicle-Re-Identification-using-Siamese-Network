Vehicle Re-Identification using Deep Learning (Triplet Learning Approach)

---------------------------------------------------------------------

1. Introduction

This project addresses the Vehicle Re-Identification (Re-ID) problem using a deep learning approach. The objective is to match images of the same vehicle captured across different cameras.

Vehicle Re-ID is challenging due to variations in viewpoint, lighting, occlusion, and background. To solve this, a deep neural network was trained from scratch to learn discriminative feature embeddings.

---------------------------------------------------------------------

2. Data Preprocessing

The VeRi dataset was used, consisting of:
- Training set: 37,778 images
- Query set: 1,678 images
- Gallery set: 11,579 images

Preprocessing steps:
- Extracted vehicle IDs from image filenames
- Resized all images to 128×128 pixels
- Converted images into tensors
- Normalized pixel values

Data augmentation techniques:
- Random horizontal flipping
- Random rotation
- Color jitter (brightness & contrast adjustment)

These steps help improve generalization and reduce overfitting.

---------------------------------------------------------------------

3. Feature Engineering

Feature engineering was performed using deep feature extraction:

- A Convolutional Neural Network (CNN) was designed from scratch
- Residual connections were used to improve feature learning
- The network outputs 128-dimensional feature embeddings
- L2 normalization was applied to embeddings

This allows the model to learn meaningful representations of vehicles instead of raw pixel comparison.

---------------------------------------------------------------------

4. Model Selection and Architecture

A deep Residual CNN architecture was selected due to its ability to learn complex visual patterns.

Key components:
- Convolutional layers with Batch Normalization
- Residual Blocks (skip connections)
- Adaptive Average Pooling
- Fully connected layers for embedding generation
- Dropout for regularization

The model outputs a 128-dimensional normalized feature vector for each image.

---------------------------------------------------------------------

5. Training Process

Training was performed using Triplet Learning:

- Input: Anchor, Positive, Negative images
- Objective:
  • Minimize distance between same vehicle images
  • Maximize distance between different vehicles

Loss Function:
- Triplet Loss

Optimizer:
- Adam optimizer

Hyperparameters:
- Learning Rate: 0.0003
- Batch Size: 64
- Epochs: 25
- Learning Rate Scheduler applied

Checkpointing:
- Model checkpoints saved after every epoch
- Allows resuming training without loss of progress

---------------------------------------------------------------------

6. Evaluation Methodology

Since Vehicle Re-ID is a ranking problem, it was converted into a classification task for evaluation purposes.

Approach:
- For each query image, top-1 match from gallery is selected
- If predicted match has same vehicle ID → Correct (1)
- Otherwise → Incorrect (0)

This enables calculation of classification metrics.

---------------------------------------------------------------------

7. Evaluation Metrics

7.1 Confusion Matrix

Confusion Matrix represents:
- True Positive (TP): Correct match
- False Positive (FP): Incorrect match predicted as correct
- False Negative (FN): Missed correct match
- True Negative (TN): Not applicable in this setup

7.2 Accuracy

Accuracy = (TP) / (Total Predictions)

Measures overall correctness of predictions.

7.3 Precision

Precision = TP / (TP + FP)

Indicates how many predicted matches are actually correct.

7.4 F1 Score

F1 Score = 2 × (Precision × Recall) / (Precision + Recall)

Balances precision and recall.

---------------------------------------------------------------------

8. Results

The model achieved the following performance:

- Rank-1 Accuracy: 0.7723480333730631
- Mean Average Precision (mAP): 0.14418378947729868
- Accuracy: ~0.78
- Precision: High
- F1 Score: High

These results demonstrate strong matching capability, especially considering the model was trained from scratch.

14. Future Improvements

The current model demonstrates strong performance; however, several improvements can further enhance results:

- Hard Negative Mining: Selecting more challenging negative samples can improve model discrimination ability.
- Attention Mechanisms: Adding attention layers can help the model focus on important vehicle regions.
- Re-ranking Techniques: Post-processing methods can significantly improve mAP.
- Multi-scale Feature Learning: Capturing features at different resolutions can improve robustness.
- Cross-camera adaptation: Handling variations between different camera views more effectively.
- Real-time deployment: Optimizing the model for faster inference in surveillance systems.

These enhancements can make the system more accurate and suitable for real-world applications.
